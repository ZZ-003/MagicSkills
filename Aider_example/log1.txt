AST（抽象语法树）核心知识，适合用于程序分析、安全研究或编译器理解：                                                             

--------------------------------------------------------------------------------------------------------------------------------

🌲 1. 什么是 AST？                                                                                                              

 • AST（Abstract Syntax Tree） 是源代码的结构化表示，忽略空格、注释等无关细节。                                                 
 • 它反映程序的语法结构，而非运行时行为。                                                                                       
 • 广泛用于：编译器、静态分析、漏洞检测、代码重构等。                                                                           

--------------------------------------------------------------------------------------------------------------------------------

🔧 2. 本工具使用的 AST 框架                                                                                                     

 • 基于 Python 库 pycparser（支持 C89/C99 子集）。                                                                              
 • 输出格式为文本树形结构，通过 ast.show() 打印。                                                                               
 • 根节点始终是 FileAST。                                                                                                       

--------------------------------------------------------------------------------------------------------------------------------

🏗️ 3. 核心 AST 节点类型                                                                                                         

                                                           
 节点类型  用途                 示例                       
 ───────────────────────────────────────────────────────────────────────────── 
 FileAST                       整个 C 文件的根节点  包含所有全局声明和函数定义 
 FuncDef                       函数定义             int main() { ... }         
 Decl                          变量/函数声明        int x = 5;                 
 TypeDecl, PtrDecl, ArrayDecl  类型信息             int* p;, char buf[10];     
 Compound                      代码块 { ... }       函数体或 if/while 块       
 Assignment                    赋值语句             a = b + 1;                 
 If, While, For                控制流               if (x>0) { ... }           
 ID                            标识符（变量名）     x, printf                  
 Constant                      字面量               42, "hello"                
 BinaryOp                      二元运算             a + b, x == 0              
 FuncCall                      函数调用             puts("hi");                
                                                                               

--------------------------------------------------------------------------------------------------------------------------------

🔍 4. 典型分析场景                                                                                                              

✅ 检测危险函数调用                                                                                                             

查找 FuncCall → ID(name)，如：                                                                                                  

                                                                                                                                
FuncCall:                                                                                                                       
  ID: gets                                                                                                                      
                                                                                                                                

可用于识别 gets, strcpy 等不安全函数。                                                                                          

✅ 构建控制流图（CFG）                                                                                                          

从 If, While, For, Return 等节点提取控制转移逻辑。                                                                              

✅ 内存操作分析                                                                                                                 

关注：                                                                                                                          

 • FuncCall 到 malloc/free                                                                                                      
 • PtrDecl（指针声明）                                                                                                          
 • UnaryOp: *（解引用）                                                                                                         

--------------------------------------------------------------------------------------------------------------------------------

⚠️ 5. 限制与注意事项                                                                                                            

 • 不支持 GCC 特有扩展（如 __attribute__）。                                                                                    
 • 宏需预处理：复杂宏可能被展开失败。                                                                                           
 • 无跨文件符号解析：仅分析单个 .c 文件。                                                                                       
 • 类型未完全解析：仅保留语法层面的类型信息。                                                                                   

--------------------------------------------------------------------------------------------------------------------------------
