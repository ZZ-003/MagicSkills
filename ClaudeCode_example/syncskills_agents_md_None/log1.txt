● AST（抽象语法树）是程序源代码的结构化表示，它捕捉了代码的语法层次，忽略了格式、注释和空格等细节。下面我结合技能文档为你系统介绍   
  AST 的相关知识：

  1. AST 是什么？

  - 抽象语法树：将源代码转换为树状数据结构，每个节点代表语言中的一个构造（如函数、变量、表达式）
  - 与具体语法树的区别：AST 不包含分号、括号等语法细节，只关注程序结构
  - 核心特性：
    - 表示语法结构，而非运行时行为
    - 保留声明、语句、表达式的层次关系
    - 便于程序分析和变换

  2. AST 的主要应用场景

  ┌───────────────┬──────────────────────────────────────────────────┐
  │     领域      │                     具体用途                     │
  ├───────────────┼──────────────────────────────────────────────────┤
  │ 编译器/解释器 │ 语法分析后生成 AST，用于语义分析、优化、代码生成 │
  ├───────────────┼──────────────────────────────────────────────────┤
  │ 静态分析工具  │ 检测代码质量、安全漏洞、编码规范违规             │

  3. C 语言 AST 的典型结构（基于 pycparser）

  FileAST（根节点，整个编译单元）
  ├── Decl（全局变量声明）
  ├── FuncDef（函数定义）
  │   ├── Decl（函数签名）
  │   ├── ParamList（参数列表）
  │   └── Compound（函数体）
  └── ...

  关键节点类型：

  - FileAST：AST 的根，包含所有顶级声明
  - FuncDef：完整的函数定义
  - Decl：变量/函数/参数声明
  - TypeDecl/PtrDecl/ArrayDecl：类型声明（int、指针、数组）
  - Compound：代码块 { ... }
  - Assignment/Return/If/While/For：语句节点
  - ID/Constant/BinaryOp/FuncCall：表达式节点

  4. 如何用这个技能生成 AST？

  这个 c_2_ast 技能可以帮你将 C 代码转换为 AST：

  操作步骤：

  1. 准备 C 代码：提供 .c 文件路径或直接粘贴 C 代码
  2. 运行解析脚本：
  python3 c_2_ast.py --input your_code.c
  3. 查看结果：输出为层次化的文本 AST，显示完整的语法结构

  示例输出片段：

  FileAST:
  FuncDef:
    Decl: main
    FuncDecl:
      TypeDecl:
        IdentifierType: ['int']
    Compound:
      FuncCall:
        ID: printf
        ExprList:
          Constant: string, "Hello"
      Return:
        Constant: int, 0

  5. AST 的深入用途

  - 控制流图（CFG）构建：基于 If、While、For 等节点分析程序执行路径
  - 数据流分析：跟踪变量定义和使用，检测未初始化变量等
  - 模式匹配：识别特定代码模式（如缓冲区溢出风险）
  - 代码转换：在 AST 层面进行重构或优化
