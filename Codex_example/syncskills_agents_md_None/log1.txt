• 这里的 AST 通常指 Abstract Syntax Tree，即“抽象语法树”。它把代码表示成树形结构，保留“语法关系”，但去掉空格、
  注释、缩进这类不影响语法本质的内容。

  比如这段 C 代码：

  int add(int a, int b) {
      return a + b;
  }

  可以抽象理解成：

  FileAST
  └── FuncDef
      ├── Decl: add
      ├── ParamList
      │   ├── Decl: a
      │   └── Decl: b
      └── Compound
          └── Return
              └── BinaryOp: +
                  ├── ID: a
                  └── ID: b

  你可以先抓住这几个核心点：

  - AST 表示的是“代码结构”，不是运行结果。
  - 根节点通常是整个文件，比如 FileAST。
  - 常见节点有：
      - FuncDef：函数定义
      - Decl：声明
      - Compound：代码块 { ... }
      - If / For / While：控制流
      - Assignment：赋值
      - Return：返回
      - BinaryOp：二元运算
      - FuncCall：函数调用
      - ID / Constant：变量和常量

  AST 为什么重要：

  - 编译器前端会先把源码转成 AST
  - 静态分析工具靠 AST 理解代码结构
  - 漏洞检测常用 AST 找危险函数、控制流、指针操作
  - 后续还能从 AST 构造 CFG、DFG

  要注意的是，AST 只解决“语法结构”问题，不直接告诉你：

  - 变量最终指向什么
  - 跨文件符号怎么解析
  - 程序运行时会发生什么
