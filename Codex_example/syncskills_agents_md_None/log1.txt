• AST 是 Abstract Syntax Tree，中文通常叫“抽象语法树”。你可以把它理解成：把代码从“文本”变成“结
  构”。

  比如这段 C 代码：

  int add(int a, int b) {
      return a + b;
  }

  它的 AST 可以粗略看成：

  FileAST
  └─ FuncDef(add)
     ├─ ParamList
     │  ├─ Decl(a:int)
     │  └─ Decl(b:int)
     └─ Compound
        └─ Return
           └─ BinaryOp(+)
              ├─ ID(a)
              └─ ID(b)

  核心理解

  - AST 关心“结构”，不关心空格、注释、缩进这些表面形式。
  - AST 比“语法分析树”更抽象，去掉了很多语法细节，更适合程序分析。
  - AST 描述的是“代码怎么写的”，不是“代码运行后会怎样”。

  常见节点

  - FileAST：整个文件
  - FuncDef：函数定义
  - Decl：声明
  - Compound：代码块 { ... }
  - If / For / While：控制流
  - Assignment：赋值
  - Return：返回
  - BinaryOp：二元运算，如 a + b
  - FuncCall：函数调用
  - ID：变量名
  - Constant：常量

  AST 有什么用

  - 编译器前端
  - 静态分析
  - 漏洞检测
  - 代码重构
  - 代码搜索/相似度分析
  - 生成 CFG、DFG 等中间表示

  它的局限

  - AST 只告诉你“语法结构”
  - 不直接告诉你变量真实类型、跨文件引用、运行时值
  - 想做更深分析，通常还要配合符号表、类型分析、CFG、DFG
