• AST 是 Abstract Syntax Tree，中文通常叫“抽象语法树”。它把源码表示成一棵树，重点保留“结构”和“语
  义相关的语法信息”，而不是源码的原始写法。

  比如这段 C 代码：

  int add(int a, int b) {
      return a + b;
  }

  可以抽象成类似这样的结构：

  FileAST
  └── FuncDef
      ├── Decl: add
      ├── ParamList
      │   ├── Decl: a
      │   └── Decl: b
      └── Compound
          └── Return
              └── BinaryOp: +
                  ├── ID: a
                  └── ID: b

  你可以把 AST 理解成：代码不再是一串文本，而是“函数定义、参数、返回语句、加法表达式”这些结构化
  对象。

  AST 的核心特点

  - 它表示的是语法结构，不是运行结果。
  - 它通常忽略空格、注释、缩进这些格式信息。
  - 它比原始解析树更简洁，所以叫“抽象”语法树。
  - 它很适合做静态分析、编译器前端、代码检查和安全分析。

  AST 和解析树的区别

  - 解析树（Parse Tree / CST）更贴近文法规则，细节更多。
  - AST 会去掉很多中间层和无关符号，只保留对理解代码结构真正重要的部分。
  - 简单说：解析树更“忠于语法”，AST 更“忠于结构”。

  AST 里常见的节点

  - FileAST / Module：整份源码
  - FuncDef / FunctionDef：函数定义
  - Decl：声明
  - Compound / Block：代码块
  - If / For / While：控制流
  - Assignment：赋值
  - Return：返回
  - BinaryOp：二元运算，比如 +、==
  - ID / Name：变量名
  - Constant / Literal：常量

  AST 能做什么

  - 编译器前端：理解源码结构
  - 静态分析：找 bug、查漏洞、做规则检测
  - 代码重构：批量改函数名、插入日志、自动迁移 API
  - 安全研究：识别危险函数调用、控制流模式、内存操作
  - 构建 CFG/DFG：进一步做控制流图、数据流图分析

  学习 AST 最值得先搞懂的 5 个点

  1. 词法分析 -> 语法分析 -> AST 的流程
  2. 一段代码如何映射成树节点
  3. 表达式、语句、声明这三类节点的区别
  4. AST 只描述“写了什么”，不直接表示“怎么执行”
  5. AST 之后还可以继续做语义分析、CFG、DFG

  建议的学习路线

  1. 先理解 AST 是“代码的结构化表示”
  2. 找几段简单代码，手动画 AST
  3. 用工具实际打印 AST
  4. 学会遍历 AST：前序、递归访问、节点匹配
  5. 再进入进阶：符号表、类型检查、CFG、DFG、污点分析
