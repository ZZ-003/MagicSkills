• AST 是 Abstract Syntax Tree，中文一般叫“抽象语法树”。它是把代码按“语法结构”组织成一棵树，而不是
  按源文件文本逐字符保存。

  比如这行代码：

  return a + b * 2;

  AST 更接近这样：

  Return
    BinaryOp(+)
      ID(a)
      BinaryOp(*)
        ID(b)
        Constant(2)

  这里能直接看出乘法先于加法，这就是 AST 的价值: 它把“结构”和“优先级”显式表示出来。

  你可以先抓住这几个核心点：

  - Token 是词法层，像 return、a、+、2
  - AST 是语法层，表示“谁和谁组成什么结构”
  - AST 会忽略很多不重要的表面细节，比如空格、注释、部分括号形式
  - 它通常比完整语法分析树更简洁，所以叫“抽象”语法树

  AST 常见用途：

  - 编译器前端
  - 静态分析
  - IDE 功能，比如跳转、重构、补全
  - 代码格式化和代码转换
  - 安全分析，比如识别危险函数调用、控制流结构、指针操作

  常见节点类型：

  - FileAST / Module: 整个文件
  - FuncDef / FunctionDef: 函数定义
  - Decl: 变量或函数声明
  - If / For / While: 控制流
  - Assignment: 赋值
  - Return: 返回
  - BinaryOp: 二元运算
  - FuncCall: 函数调用
  - ID / Name: 标识符
  - Constant: 常量

  AST 的局限也要知道：

  - 它描述的是“语法”，不是“运行时行为”
